# 开发路线图

## Phase 1: 项目初始化与技术选型

目标：确定 PySide6 桌面端优先方案，完成项目骨架、文档、基础页面壳。

产出：

- PRD、技术方案、页面结构、数据模型。
- Python 包结构、桌面入口、SQLite schema、服务层骨架。

优先级：P0。

依赖：无。

## Phase 2: 八字排盘基础能力

目标：替换当前占位算法，接入可靠历法、节气、干支、十神、藏干、真太阳时规则。

当前进展：

- 已实现干支年、近似节气定月、儒略日干支日、日干起时。
- 已实现四柱天干十神关系、五行分布、阴阳分布。
- 已覆盖立春切年、小寒前后月支、已知四柱样例等测试。

产出：

- 农历/阳历转换（待接入）。
- 精确节气定月（当前为固定日期近似）。
- 四柱、十神、藏干、纳音扩展位（纳音待补）。
- 排盘校验用例（持续扩充）。

优先级：P0。

依赖：Phase 1。

## Phase 3: 基础解读与双模式结果展示

目标：让结果页可用于真实阅读。

产出：

- 专业术语模式。
- 初学者易懂模式。
- 五行图表、阴阳比例、折叠分析区。
- 解释模板版本化。

优先级：P0。

依赖：Phase 2。

## Phase 4: AI 问命对话

目标：AI 读取结构化盘面和基础解释，提供咨询式对话。

当前进展：

- 已实现离线降级、OpenAI-compatible 环境变量适配、对话保存和历史打开。
- 已实现 AI 会话 Markdown 对话记录导出。

产出：

- AI provider 设置（当前通过环境变量）。
- Prompt 模板。
- 会话保存。
- 无 AI 配置时的离线降级。

优先级：P1。

依赖：Phase 2、Phase 3。

## Phase 5: 黄历择日

目标：支持按用途筛选候选日期。

当前进展：

- 已实现启发式评分、范围校验、结果保存、历史打开和 Markdown 候选表导出。

产出：

- 黄历规则表。
- 宜忌、冲煞、用途评分。
- 日期范围筛选与结果解释。

优先级：P1。

依赖：Phase 1，部分依赖历法能力。

## Phase 6: 历史记录与导出

目标：让个人研究可以长期复查。

当前进展：

- 已实现历史列表、类型筛选、关键词搜索、详情预览、打开、删除和 Markdown 导出。
- 已实现用户资料保存、载入编辑、列表和删除。

产出：

- 历史记录详情页。
- Markdown/PDF/PNG 导出（Markdown 已实现，PDF/PNG 待接入）。
- 报告模板。
- 本地备份/恢复扩展位。

优先级：P1。

依赖：Phase 2、Phase 3。

## Phase 7: 优化与扩展

目标：提升正式软件体验并准备扩展模块。

产出：

- 用户资料管理完善。
- 紫微、六爻、梅花、奇门入口。
- 云同步和案例库预留。
- 打包发布流程。

优先级：P2。

依赖：前置阶段稳定。
